개발자: 이웅주(CxImage 사용, 스킨 기능 추가)


MFC와 CxImage의 정적 라이브러리를 사용함(DLL 필요 없음)


※ 알림:

- CxImage 5.99c를 사용했다.

- 라이브러리 파일 용량을 줄이기 위해 TIFF만 사용할 수 있도록 되어 있으며 파일로 쓰기 또한 제거되어 있다. 설정 변경은
ximcfg.h에서 한다.



※ 참조:

CxImage를 정적 라이브러리 버전으로 컴파일하여 런처에 링킹하기

1.	런처에서 압축을 풀기 위해 사용하는 오픈 소스는 이미지의 압축을 풀때도 사용된다. 그런데 전역으로 사용한 저작권 문자열의 
	변수 명이 같아 LNK2005 에러를 발생하므로 링킹이 되지 않는다. 따라서 CxImage 프로젝트에 포함된 zlib에서 _deflate_copyright
	와 _inflate_copyright를 제거한다(어차피 사용되지 않는다). 


2.	CxImage 솔루션을 열고 Zlib, JPEG, TIFF 프로젝트를 다음 옵션을 주고 컴파일한다.

	...	일반 > 구성형식: 정적 라이브러리, 일반 > MFC 사용: 정적 라이브러리 사용

	
3.	CxImage를 컴파일하되(접미어로 DLL붙은 것들은 아님) 라이브러리 > 추가종속성에 위에 컴파일한 zlib.lib, jpeg.lib, tiff.lib
	를 입력하고 컴파일한다. 경고는 무시한다.

4.	이제 생성된 lib 파일을 런처에 링킹하면 DLL을 사용하지 않는다. 

// TRUE값이 이진 형태 플래그이다.
(사용 예) g_pFtp->SetupExtendDownload( temp2, ".\\", TRUE );
