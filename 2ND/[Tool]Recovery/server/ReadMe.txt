08/03/06

빌드 넘버 자동화

참조: http://www.codeproject.com/KB/dotnet/build_versioning.aspx

빌드 전 이벤트, 후 이벤트를 사용하여 서버 빌드 번호를 자동화한다. 
클라이언트는 컴파일 시에 version.h 파일을 열어서 상수로 참조할 수 있으며,
이를 통해 서버와 버전 비교를 할 수 있다.

서버 버전이 바뀌면 클라이언트는 무조건 다시 컴파일해야한다. 서버 변동 시 동기화를
맞추기 위해서이다. 잦은 서버 변경을 막기 위해 클라이언트와 서버는 [메이저 번호][마이너 번호]만
비교한다. 빌드 번호는 무시할 것이다. 따라서 서버를 컴파일할 경우 반드시 클라이언트의 변경 내역이
없다고 해도 다시 컴파일해야한다.

클라이언트를 컴파일하면 [빌드 번호]만 카운팅되고, 서버를 컴파일하면 [마이너 번호]가 카운팅된다.

VerCopy.exe				프로그램의 버전 정보를 추출해서 복사한다
VerHeader.exe			헤더 파일을 갱신한다
RCStamp.exe				리소스 파일의 FILEVERSION을 갱신한다
VerCopyFromHeader.exe	헤더 파일에서 버전 정보를 추출해서 복사한다
